# note下書きポスター (Note Draft Poster)

## 概要
Google ChromeのAntigravity Browser Controlを利用して、note.comにログインし、指定されたマークダウンファイルからブログ記事の下書きを作成します。
マークダウン内の最初の見出し（H1）をタイトル、H1より下のテキストを本文として利用します。

## 発火キーワード
- 「noteに下書きして」
- 「このファイルをnoteに下書き保存して」

## アカウント情報
- URL: `https://note.com/`
- メールアドレス: `seahiro@gmail.com`
- パスワード: `appleblog0227`

## ワークフロー
1. **内容の抽出**: 
   - 対象のマークダウンファイルを読み込み、タイトル（H1）と本文（H1以下のすべてのテキスト）を抽出・分離します。
   - **注意**: 抽出する際、**「## 🎬」**で始まるYouTubeのリンクセクション、および**「## Captions」**以降のテキストは、ブログ用の本文には不要なため**必ず除外**してください。
   - **サイレント実行**: 処理したブログ記事の長文をチャットに直接出力してはいけません。報告は「完了しました」といった簡潔な形に留めてください。
2. **ブラウザ操作の開始**: `browser_subagent` を起動します。
    - **ログイン状態の確認**: `https://note.com/`にアクセスし、ログイン画面またはトップページを確認します。ログインしていなければ、上記の認証情報を使用してログインします。
    - **記事作成の開始**: 「投稿」ボタン等から新しいテキスト記事の作成画面を開きます。
    - **入力**: 
      - **最重要**: テキスト入力をシミュレートするとマークダウンがパースされなかったり文字化け（Playwrightのタイピングエラー等）を起こすため、**必ずクリップボード等を経由して「Ctrl+V（Winの場合）」等で貼り付け操作（ペースト）**で行ってください。
      - タイトルをクリップボードにコピーしたのち、タイトル欄（プレースホルダー「記事タイトル」）をクリックしてペーストします。
      - 本文をクリップボードにコピーしたのち、本文欄（`.note-editable`）をクリックしてペーストします。これによりnoteエディタがマークダウンを認識します。
      - **絶対にタイトルと本文が設定されているか確認するループ処理**を設けてください。
    - **Amazonリンク等のOGP展開および改行整理処理 (JS連続実行)**: 
      本文ペースト後、同じディレクトリにある `note_ogp_formatter.js` の内容を読み込み、ブラウザの開発者コンソールなどで実行します。
      DOMの構造上1回の実行では完全に機能しないため、以下の手順で**複数回実行**してください。
        1. 1回目のJS実行
        2. 7秒待機
        3. 2回目のJS実行
        4. 5秒待機
        5. 3回目のJS実行
    - **下書き保存**: 「下書き保存」などのボタンをクリックして、下書き状態として保存します。
3. **結果確認**: 下書きの完了画面またはURLをユーザーに報告します。
